Installation

Command Line

Install Anywhere (Standalone)

Just like a wig install, you can install Sass on Windows, Mac, or Linux by downloading the package for your operating system from GitHub and adding it to your PATH.

Install on Windows (Chocolatey)

If you use the Chocolatey package manager for Windows, you can install Sass by running 
                
                    choco install sass

Install on Mac OS X or Linux (Homebrew)

If you use the Homebrew package manager for Mac OS X or Linux, you can install Sass by running 
                
                    brew install sass/sass/sass

Sass is a preprocessor scripting language that provides you with features that don't exist in CSS yet like nesting, mixins, inheritance, and other nifty goodies that help you write robust, maintainable CSS.

Two Syntaxes

    .scss - most common
    .sass - more unusual, indented syntax
.scss .sass 
                            
.container {
                            
    float: left;
                            
    width: 100%;
                            
    p {
                            
        color: #333;
                            
    }
                            
}
                            

                            
.container
                            
    float: left
                            
    width: 100%
                            
p
                            
   color: #333
SASS CANNOT be read/understood by browsers To make this work, you must compile sass into css by using this command: 
                
                    sass --watch input.scss output.css
As long as the Sass file is being "watched" your changes will automatically compile into the output.css file.

Partials

"partial" sass files contain:

    Little snippets of CSS which can be included in other Sass files
    Modularize your CSS, keeping things easier to maintain
    Named with a leading underscore 
                    
                        _partial.scss
    For instance, if I wrote @import _partials.scss or @use _partials.css (functions you can read about in the at-rules section) in the beginning of my main scss file, then I could reference functions, variables, and classes from _partials.scss in my main scss file.

Variables

Sass allows for the use of variables, which can reference colors, numbers, strings, etc.

    Use the $ to declare and reference a variable
                
                    $logo-color: #ff00bb
                

                ...which can then be referenced as:

                
                    button {
                        background: $logo-color;
                    }

                    h2 {
                        color: $logo-color;
                    }
    Sass variables are imperative, which means if you use a variable and then change its value, the earlier use will stay the same.
    CSS variables are declarative, which means if you change the value, it’ll affect both earlier uses and later uses.
    Variables that are defined by a built-in module cannot be modified.

Nesting

    With Sass, you can nest your CSS selectors in a way that follows the same visual hierarchy of your HTML.
    HOWEVER! Overly nested rules will result in overqualified CSS which can cause complications in the compilation of your Sass file.


For example: If you have multiple buttons, you can combine all of them under one class in Sass and have sub classes for each type button
Either use indent or & 
                        
                            .button {
                                .sucess {
                                    color: green;
                                }
                            }
                        
                    
                        
                            .btn {
                                &:focus {}
                                &:hover {}
                                &:active {}
                            }

Interpolation

Interpolation can be used almost anywhere in a Sass stylesheet to embed the result of a SassScript expression into a chunk of CSS. Just wrap an expression in #{} in any of the following places:

SassScript

Interpolation can be used in SassScript to inject SassScript into unquoted strings. This is particularly useful when dynamically generating names (for example for animations), or when using slash-separated values. Note that interpolation in SassScript always returns an unquoted string.

Quoted Strings

In most cases, interpolation injects the exact same text that would be used if the expression were used as a property value. But there is one exception: the quotation marks around quoted strings are removed (even if those quoted strings are in lists). This makes it possible to write quoted strings that contain syntax that’s not allowed in SassScript (like selectors) and interpolate them into style rules.

At-Rules

@MIXIN

    CSS Classes with similar styles?
    Create one MIXIN class that can be mentioned in other classes!
    For example, if I have
                    
                        @mixin example {
                            color: $logo-color;
                            size: $default-size;
                        }
    and I reference that in another class using @include
                    
                        specialText {
                            @include example;
                            margin-left: auto;
                            margin-right: 100px;
                        }
    The compiled css would be:
                    
                        specialText {
                            color: $logo-color;
                            size: $default-size;
                            @include example;
                            margin-left: auto;
                            margin-right: 100px;
                        }

@INCLUDE

    Allows user to include mixins in classes

@IMPORT

    Extends the CSS at-rule to load styles, mixins, functions, and variables from other stylesheets
    Similar to @import in Java

@USE

    Newer version of @import
    Only imports desired module ONCE
    Allows user to keep track of variables
    For example, let's say we have the variable $primary-color in a partial file that we reference in our main scss file.
    HOWEVER, we also have a variable named $primary-color in our main scss file!
    To avoid errors, you would do @use exampleFile.scss as e (the "e" can be whatever you want it to be)
    Then, you can reference the partial file variable as e.$primary-color which will differentiate it from the $primary-color in our main file

@FORWARD

    Can be used prior to @use
    "Forwards" mixins, functions, and variables from a stylesheet that are included in another stylesheet

@FUNCTION

    defines custom functions that can be used in SassScript expressions

@EXTEND

    Allows selectors to inherit styles from one another
    Different from @mixin

@AT-ROOT

    Puts styles within Sass document at the root of the CSS document
    Not commonly used
    Used for advanced nesting

@ERROR

    Causes compilation to fail with an error message
    Include the line @error [insert error message here]; to throw an error and output said message in terminal which will stop compilation

@WARN

    Prints a warning without stopping compilation entirely
    Include the line @warn [insert warn message here]; to throw a warning and output said message in terminal but it won't prevent compilation

@DEBUG

    Prints a message for debugging purposes
    Can be used to double check values during compilation
    For example, if $primary-color has the value #c69, the line @debug "color is $primary-color"; will print "color is #c69" in terminal during compilation

Flow Control

Flow control rules like @if, @each, @for, and @while control whether or how many times styles are emitted.
@if controls whether or not a block is evaluated. @each evaluates a block for each element in a list or each pair in a map. @for evaluates a block a certain number of times. @while evaluates a block until a certain condition is met.

Operators

Sass supports a handful of useful operators for working with different values. These include the standard mathematical operators like + and *, as well as operators for various other types:

  • == and != are used to check if two values are the same.
  • +, -, *, /, and % have their usual mathematical meaning for numbers, with special behaviors for units that matches the use of units in scientific math.
  • <, <=, >, and >= check whether two numbers are greater or less than one another.
  • and, or, and not have the usual boolean behavior. Sass considers every value “true” except for false and null.
  • +, -, and / can be used to concatenate strings.

    If I have the following code in my file:

    <p class="paragraph-1">Sassy Slay</p>
    <p class="paragraph-2">Sassy Slay</p>
    <p class="paragraph-3">Sassy Slay</p>
    <p class="paragraph-4">Sassy Slay</p>
    <p class="paragraph-5">Sassy Slay</p>

    I can use the following code: 

    @for $i from 1 to 6
                        
        {
                        
            .paragraph-#{$i}
                        
        {
                        
            font-size: 10px*$i;
                        
            @if $i%2==0 {
                        
                background-color: rgb(250, 202, 209)
                        
            }
                        
            @else {
                        
                background-color: rgb(184, 226, 239)
                        
            }
                        
        }
                        
    }
    To produce this:

    Sassy Slay

    Sassy Slay

    Sassy Slay

    Sassy Slay

    Sassy Slay

    If you look closely, each "paragraph" class has a number, and the for loop uses a variable "i", which references the number of each paragraph class. The for loop iterates through numbers 1 through 6 (exclusive), multiplies the font size by the paragraph number, makes the background pink if the number is even, and the background blue if it's odd. Of course, this is a very primitive example of for loops, but the possibilities with functions and loops are endless.

    Modules

    Sass provides many built-in modules which contain useful functions (and the occasional mixin). These modules can be loaded with the @use rule like any user-defined stylesheet, and their functions can be called like any other module member. All built-in module URLs begin with sass: to indicate that they're part of Sass itself.

    Sass provides the following built-in modules:
  • sass:math - provides functions that operate on numbers.
  • sass:string - makes it easy to combine, search, or split apart strings.
  • sass:color - generates new colors based on existing ones, making it easy to build color themes.
  • sass:list - lets you access and modify values in lists.
  • sass:map - makes it possible to look up the value associated with a key in a map, and much more.
  • sass:selector - provides access to Sass’s powerful selector engine.
  • sass:meta - exposes the details of Sass’s inner workings.
